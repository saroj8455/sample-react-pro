# React + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react/README.md) uses [Babel](https://babeljs.io/) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

Hereâ€™s a list of reusable components commonly used in React applications, along with a brief description of their purpose and scenarios where they can be applied.

---

### **1. Button**

- **Purpose**: Triggers an action or event.
- **Features**: Different styles (primary, secondary, danger), sizes, icons, and states (loading, disabled).
- **Example Use Case**: Submit forms, trigger modals, or perform API actions.

---

### **2. Input Field**

- **Purpose**: Captures user text input.
- **Features**: Validation, placeholder, error messages, icon support, password visibility toggle.
- **Example Use Case**: Login forms, search bars, filters.

---

### **3. Dropdown / Select**

- **Purpose**: Provides a list of selectable options.
- **Features**: Multi-select, search/filter, grouping, custom rendering.
- **Example Use Case**: Selecting a country, filtering categories, or choosing tags.

---

### **4. Modal / Dialog**

- **Purpose**: Displays overlay content such as forms, confirmations, or alerts.
- **Features**: Closable via button or backdrop click, header/footer slots, animations.
- **Example Use Case**: Confirmation dialogs, detailed views, or embedded forms.

---

### **5. Card**

- **Purpose**: Displays grouped information in a styled container.
- **Features**: Header, body, and footer sections; customizable actions like buttons or icons.
- **Example Use Case**: Product listings, user profiles, or dashboard widgets.

---

### **6. Table**

- **Purpose**: Displays tabular data.
- **Features**: Pagination, sorting, filtering, expandable rows, custom cell rendering.
- **Example Use Case**: Admin dashboards, reporting, or inventory systems.

---

### **7. Loader / Spinner**

- **Purpose**: Indicates loading states.
- **Features**: Variants like spinners, progress bars, skeleton loaders.
- **Example Use Case**: Loading API data, waiting for file uploads.

---

### **8. Tabs**

- **Purpose**: Provides a way to navigate through different views in the same space.
- **Features**: Active tab indication, scrollable tabs, lazy-loading of tab content.
- **Example Use Case**: Profile sections, dashboard views, or settings pages.

---

### **9. Notification / Toast**

- **Purpose**: Shows temporary feedback or alerts.
- **Features**: Configurable duration, types (success, error, info), closable.
- **Example Use Case**: Form submission success, error alerts.

---

### **10. Breadcrumb**

- **Purpose**: Displays navigation paths.
- **Features**: Customizable separators, active state, clickable links.
- **Example Use Case**: Website navigation, hierarchical data structures.

---

### **11. Pagination**

- **Purpose**: Breaks content into manageable pages.
- **Features**: Next/Previous buttons, jump to page, total item count display.
- **Example Use Case**: Blogs, product listings, or API data.

---

### **12. Accordion / Collapsible Panel**

- **Purpose**: Displays expandable/collapsible sections.
- **Features**: Multiple or single expansion, lazy-loaded content.
- **Example Use Case**: FAQs, data filters, or detailed views.

---

### **13. Tooltip**

- **Purpose**: Displays additional information on hover or focus.
- **Features**: Custom placement, animations, and delays.
- **Example Use Case**: Icon explanations, button hints.

---

### **14. Navbar / Sidebar**

- **Purpose**: Provides site navigation.
- **Features**: Responsive layouts, dropdown menus, collapsible sections.
- **Example Use Case**: Website headers, admin dashboards.

---

### **15. Avatar**

- **Purpose**: Displays user profile pictures or placeholders.
- **Features**: Support for initials, images, sizes, and status indicators.
- **Example Use Case**: Chat applications, user lists, or dashboards.

---

### **16. Slider / Range Selector**

- **Purpose**: Allows selection within a range.
- **Features**: Single or dual-handle sliders, tooltips, step increments.
- **Example Use Case**: Price filters, audio volume controls.

---

### **17. Date Picker**

- **Purpose**: Captures date input from users.
- **Features**: Range selection, disabled dates, formats, and localization.
- **Example Use Case**: Booking systems, forms, or event scheduling.

---

### **18. Chip / Tag**

- **Purpose**: Displays items like selected filters or categories.
- **Features**: Closable tags, customizable colors, and icons.
- **Example Use Case**: Selected filters, user roles.

---

### **19. Carousel**

- **Purpose**: Displays a slideshow of items.
- **Features**: Automatic scrolling, navigation buttons, indicators.
- **Example Use Case**: Image galleries, testimonials.

---

### **20. Alert / Message**

- **Purpose**: Displays important information to users.
- **Features**: Dismissable alerts, severity levels (info, success, error, warning).
- **Example Use Case**: Form errors, system notifications.

---
